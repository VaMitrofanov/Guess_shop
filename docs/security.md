# Безопасность

Репозиторий **публичный** (`Guess_shop`). Правило: секреты, платёжные ассеты и приватные
картинки — только в env / БД, никогда в коммит.

## Что проверено

- **Секреты в коде** — не найдено. Токены (`TG_TOKEN`, `VK_TOKEN`, `.ROBLOSECURITY`,
  `DATABASE_URL`) читаются только из env.
- **TWA API** — все `/api/twa/*` роуты закрыты `extractTwaUser` (JWT `twa-admin`).
  Исключения: `auth` (точка входа) и `debug` (за `ADMIN_SECRET` + `timingSafeEqual`).
- **initData** — HMAC-проверка подписи Telegram + отклонение старше 24 ч.
- **wb-code** — атомарный резерв в транзакции, валидация формата кода.

## <a name="twa-auth"></a>Известные риски

### 1. TWA auth fallback (Path 2) — приоритет высокий
`src/app/api/twa/auth/route.ts`: если запрос без `initData`, роут доверяет `userId` из тела
и проверяет только `isAdmin(userId)`. Telegram user ID **не секретны** (видны в форвардах/чатах).
Любой, кто знает TG ID менеджера, может получить валидный 12-часовой admin-токен → полный
доступ к заказам, cookie, балансу, выкупу.

Рекомендация: убрать Path 2 (требовать `initData` всегда) либо привязать fallback к
дополнительному shared-secret. Предварительно проверить, действительно ли актуальный Telegram
не отдаёт `initData` на целевых устройствах (современные клиенты отдают) — если отдаёт,
fallback просто удаляется без потери доступа.

### 2. Rate-limiting на публичных API
`/api/wb-code` (резерв/статус) не имеет rate-limit → теоретический перебор кодов
(пространство 36^7 велико, но GET раскрывает `denomination` и `claimed`). При активном
WB-потоке добавить лимит по IP/сессии.

### 3. `debug`-роут
`/api/twa/debug` за `ADMIN_SECRET`, но светит первые 12 символов `WB_API_TOKEN`.
Диагностический — удалить из прод-сборки.

### 4. Захардкоженные IP серверов
`Dockerfile`, `bots/tg/admin/hub-system.ts`, `src/app/api/twa/system/route.ts`,
`scripts/setup-swap.sh` содержат IP серверов как fallback. Репо публичный → IP уже раскрыты.
Убрать хардкод, оставить только env (`*_BOT_HEALTH_URL`).

### 5. Историческая утечка в git
`HANDOFF.md`/`PLAN.md` содержали SSH-инструкции, Coolify-панель и UUID сервисов. Файлы
выведены из трекинга (`.gitignore`), но **остаются в истории git**. Для полной чистки —
переписать историю (`git filter-repo`) и/или ротировать любые чувствительные значения,
которые там упоминались.

## select-gamepass — валидация
`api/wb-code/select-gamepass` ре-валидирует цену и `isActive`, но не проверяет принадлежность
геймпасса нику (`creatorId`). Hand-crafted POST может привязать чужой геймпасс с совпадающей
ценой. Митигируется ручным выкупом менеджером (он видит карточку). Низкий приоритет.
